# نموذج التهديدات — WorkPass (مسودة)

## نطاق النظام

- لوحة WorkPass (متصفح)
- تطبيق العامل (موبايل / PWA)
- API مركزي (`backend/server.py` + `backend/app/platform/*`)
- تكاملات: SMTP, Stripe, Entra/Google SSO, كاميرات/RTSP, Push

## أصول حرجة

| الأصل | السرية | النزاهة | التوفر |
|-------|---------|---------|--------|
| بيانات العمال (PII) | عالية | عالية | متوسطة |
| مستندات الامتثال | عالية | عالية | متوسطة |
| جلسات وجلسات Turnstile | عالية | عالية | عالية |
| مفاتيح API / Webhooks | عالية | عالية | متوسطة |
| سجلات التدقيق | متوسطة | عالية | عالية |

## تهديدات رئيسية (STRIDE مبسّط)

| التهديد | مثال | ضوابط موجودة / مخطَّطة |
|---------|------|-------------------------|
| انتحال (S) | سرقة token | HTTPS، انتهاء الجلسة، 2FA |
| العبث (T) | تعديل سجل دخول | سجلات append-only حيث أمكن، RBAC |
| إنكار (R) | إنكار إجراء إداري | audit trail، تقارير PDF |
| كشف (I) | تسريب بين مستأجرين | `company_id` في كل استعلام |
| حرمان الخدمة (D) | flood على login | rate limit (مراجعة دورية) |
| رفع صلاحيات (E) | company-admin → superadmin | `require_roles`، فصل الأدوار |

## مخاطر متبقية (قبل عقد مؤسسي كبير)

1. **اختبار اختراق** خارجي موثّق.
2. **SAML / Keycloak** لبيئات لا تستخدم Entra/Google فقط.
3. **تشفير at-rest** لقاعدة البيانات (مسؤولية السحابة + سياسة العميل).
4. **مراجعة تبعيات** (Dependabot / SBOM).
