# سياسة تصنيف البيانات والاحتفاظ

## مستويات التصنيف

| المستوى | أمثلة في BauPass | التعامل |
|---------|------------------|---------|
| **عام** | وثائق تسويق، حالة API عامة | بدون قيود |
| **داخلي** | إعدادات غير حساسة، feature flags | موظفون مخوّلون |
| **سري** | PII عمال، أرقام تأمين، صور | تشفير نقل، RBAC، سجل تدقيق |
| **سري للغاية** | كلمات مرور، API keys, OTP seeds | vault فقط، لا logs |

## الاحتفاظ (Retention) — إرشادات

| نوع البيانات | مدة افتراضية مقترحة | ملاحظات |
|--------------|---------------------|---------|
| سجلات دخول/خروج | 2–7 سنوات (حسب قانون العميل) | قابلة للتصدير PDF |
| مستندات امتثال | حتى انتهاء العقد + 90 يوماً | أرشفة عند الطلب |
| جلسات / tokens | أيام قليلة | إبطال فوري عند الخروج |
| سجلات أتمتة | 1 سنة | تقليل في DB القديمة |
| نسخ احتياطي DB | 30–90 يوماً | حسب مزود السحابة |

## حقوق صاحب البيانات (GDPR)

- تصدير بيانات العامل (عند الطلب عبر مسؤول المستأجر).
- حذف/إخفاء بعد نهاية العلاقة التعاقدية (مع الاحتفاظ القانوني للفواتير).

## SIEM

تصدير موصى به:

- `access_logs`, `platform_events`, فشل login, webhooks فاشلة.

تنسيق: JSON lines → Azure Sentinel / Splunk / ELK (تكامل لاحق).
