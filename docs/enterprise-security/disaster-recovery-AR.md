# التعافي من الكوارث (DR) — BauPass

## أهداف

| المقياس | هدف مقترح (قابل للتفاوض في العقد) |
|---------|-----------------------------------|
| RPO (فقدان بيانات مقبول) | ≤ 24 ساعة (نسخ يومي) — ≤ 1 ساعة مع replication |
| RTO (استعادة الخدمة) | ≤ 4 ساعات سحابة مُدارة |

## نسخ احتياطي

- **PostgreSQL**: نسخ Railway/Hetzner + runbook `docs/enterprise-backup-restore-runbook.md`
- **ملفات مرفوعة**: تخزين كائنات (S3-compatible) — نسخ متزامن مع المنطقة
- **أسرار**: Key Vault / Railway variables — لا في Git

## سيناريوهات

1. **فشل منطقة سحابية** — استعادة DB من backup، إعادة توجيه DNS.
2. **حذف مستأجر بالخطأ** — استعادة نقطة زمنية + فصل `company_id`.
3. **فشل تطبيق** — rollback إصدار Docker/Helm (`docs/private-cloud-helm-AR.md`).

## اختبار DR

- **سنوياً**: استعادة DB إلى بيئة staging والتحقق من login + تقرير واحد.
- توثيق النتيجة في ticket داخلي.

## مسؤوليات

| الطبقة | المالك |
|--------|--------|
| بنية تحتية | مشغّل السحابة + فريق DevOps |
| تطبيق | فريق BauPass |
| بيانات العميل | العميل (تصدير دوري اختياري) |
