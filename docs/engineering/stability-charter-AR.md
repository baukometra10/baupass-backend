# ميثاق الاستقرار — WorkPass

## المبدأ

المنصة تحتوي **عدداً كبيراً جداً** من القدرات. خلال الفترة القادمة الأولوية لـ:

1. الاستقرار  
2. الاختبارات  
3. الأداء  
4. الحواف البرمجية  
5. مراجعة الأمان  

**وليس** إضافة مزايا سطحية جديدة بشكل مستمر.

## تعريف «جاهز للإصدار»

- [ ] `backend/tests/` خضراء في CI
- [ ] smoke يدوي: login، worker، gate، تقرير PDF واحد
- [ ] لا regressions معروفة في i18n (`scripts/audit_i18n.js`)
- [ ] مراجعة أمن: endpoints جديدة خلف `require_auth` / `require_roles`

## مناطق خطرة (تركيز مراجعة)

| المنطقة | السبب |
|---------|--------|
| `backend/server.py` | حجم + تداخل مسؤوليات |
| رفع الملفات / OCR | حجم payload، أنواع MIME |
| Webhooks كاميرات | مصادقة مفتاح |
| SSO callbacks | state، redirect URI |
| جدولة 08:00 | timezone + مستأجرون متعددون |

## مقاييس (اختياري)

- زمن `GET /api/health` &lt; 200ms
- p95 لوحة الإدارة الرئيسية &lt; 2s (بعد تسجيل الدخول)
- معدل 5xx &lt; 0.1% في الإنتاج

## استثناءات

يُسمح بميزة جديدة فقط إذا:

- تدعم **النواة** (هوية، دخول، امتثال، تقارير)، أو
- مطلوبة **تعاقدياً** لعميل محدد، مع اختبارات ووثائق.
