# ميثاق نضج تجربة المستخدم

## أهداف

1. تقليل ازدحام الشاشة — **قسم واحد = قرار واحد**
2. توحيد المكونات بين Control Pass و admin-v2
3. تحسين اللمس (44px targets) للتابلت والموبايل
4. مصطلحات حسب القطاع عبر `uiT()` + sector-config

## قواعد تصميم

- العناوين: eyebrow + h3 + فقرة مساعدة قصيرة
- الجداول: افتراضي 10–15 صف + «عرض الكل»
- الإجراءات الخطرة: تأكيد + audit
- التحميل: skeleton وليس شاشة فارغة

## أولويات الشاشات

1. لوحة Overview / قرارات اليوم
2. العمال + المستندات
3. Reporting
4. الإعدادات (تقسيم tabs)

## ما نتجنبه

- لوحات «كل شيء في صفحة واحدة» جديدة
- مصطلحات بناء فقط في النصوص الافتراضية
