BauPass auf Railway hochladen (ohne GitHub)
============================================

WICHTIG: Keinen Token aus "Project ID" oder "Client ID" verwenden!
         Die UUID im Dashboard ist oft KEIN Login-Token.

Methode A - Browser (empfohlen, 3 Befehle)
------------------------------------------
Im Cursor-Terminal nacheinander:

  cd C:\Users\u4363\Desktop\baustelle
  railway login
  railway link
  railway up --detach

Bei railway login: Browser oeffnet sich -> einloggen.
Bei railway link: Workspace -> baupass-control -> Service "web"

Pruefen:
  https://baupass-control.up.railway.app/api/health
  https://baupass-control.up.railway.app/worker-build.json


Methode B - Account Token (nur wenn Methode A nicht geht)
---------------------------------------------------------
1. railway.app -> Klick auf Profilbild -> Account Settings
2. Links: "Tokens" (NICHT OAuth Applications)
3. "Create Token" -> Name z.B. deploy -> Create
4. Token SOFORT kopieren (wird nur einmal angezeigt)

Im Terminal:

  cd C:\Users\u4363\Desktop\baustelle
  Remove-Item Env:RAILWAY_TOKEN -ErrorAction SilentlyContinue
  $env:RAILWAY_API_TOKEN = "hier_den_neuen_token_einfuegen"
  railway whoami
  railway link
  railway up --detach

Wenn "railway whoami" Ihren Namen zeigt -> Token ist ok.
Wenn "Unauthorized" -> falscher Token oder abgelaufen.
